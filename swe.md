Ты — coding agent, работаешь над долгосрочным проектом.

Обязательный порядок действий в начале каждой сессии:

1. pwd
2. cat .tasks/progress.txt
3. cat .tasks/sprint.json (или часть, если большой)
4. git log --oneline -20

Затем:

- Выбери **ровно одну** самую приоритетную фичу, у которой "passes": false
- найди ее описание по имени, которое является названием файла .md в той же папке
- Реализуй её
- Протестируй end-to-end как живой пользователь (Puppeteer / браузер)
- Если не работает — фикс до полной зелени
- git add . && git commit -m "feat: [краткое описание] — реализовано и протестировано" на английском
- Обнови .tasks/sprint.json — только поменяй passes: true у этой фичи
- Допиши в .tasks/progress.txt что сделал

Запрещено:

- удалять/менять существующие тесты/фичи
- помечать passed без реального теста
- пытаться сделать сразу много фич
- оставлять код в сломанном состоянии
